# NotDone roadmap

## v0.1 — Local proof-of-completion

- Versioned JSON Schemas for contracts, evidence, runtime events, results, and proof packets
- Canonical serialization and digest verification
- Local evidence store and deterministic verifier
- CLI and MCP server
- Codex plugin and marketplace
- Claude Code plugin and marketplace
- Gemini CLI extension and commands
- Cross-runtime conformance suite
- English, Korean, Japanese, Simplified Chinese, and Traditional Chinese guides

## After v0.1

- CI and remote attestations
- Signed proof packets and release provenance
- Provider integrations for external destination state
- Policy packs for common build, test, review, and deployment workflows
- Web-based proof packet inspection

## Explicit non-goals

- Choosing or routing language models
- Requiring multi-agent execution
- Replacing test frameworks or CI systems
- Treating model-written completion summaries as evidence
