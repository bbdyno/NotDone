# NotDone roadmap

## v0.1 — Local proof-of-completion

- [x] Versioned JSON Schemas for contracts, evidence, runtime events, results, and proof packets
- [x] Canonical serialization and digest verification
- [x] Local evidence store and deterministic verifier
- [x] CLI and MCP server
- [x] Codex plugin and marketplace
- [x] Claude Code plugin and marketplace
- [x] Gemini CLI extension and commands
- [x] Cross-runtime conformance suite
- [x] English, Korean, Japanese, Simplified Chinese, and Traditional Chinese guides
- [x] Reproducible standalone package artifacts and release automation
- [ ] Publish `notdone`, `notdone-mcp`, and the `v0.1.0` GitHub release

## After v0.1

- Signed proof packets and remote verification attestations
- Provider integrations for external destination state
- Policy packs for common build, test, review, and deployment workflows
- Web-based proof packet inspection

## Explicit non-goals

- Choosing or routing language models
- Requiring multi-agent execution
- Replacing test frameworks or CI systems
- Treating model-written completion summaries as evidence
